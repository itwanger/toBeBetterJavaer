## 派聪明真实开发集锦


Lora 微调是一个重操作。你需要准备海量、高质量的医疗领域指令微调数据集（通常是数千到数万条“问题-答案”对），这需要医学专家标注，成本很高。同时，微调本身需要 GPU 计算资源和复杂的训练技巧。

Lora 微调主要解决的是 LLM 的“说话方式” 和 “知识水平” 问题，让模型使用更专业的术语，并可能补充一些模型预训练时不知道的最新医学知识。

![](https://files.mdnice.com/user/3903/f10d51fa-b74e-4d07-8d0e-3e72ec1cf017.png)

意图识别是一个轻操作。你可以直接利用 LLM 本身的能力，用少量（几十到几百个）的标注样本就能快速实现一个效果不错的意图识别模块。

主要解决的是“用户到底想干什么” 的问题，作用于 RAG 的最前端，能从根本上改变后续的检索和生成策略。

### 第一步，定义意图：

“高血压有什么症状？”、“糖尿病的成因是什么？”

“阿司匹林有什么副作用？”、“我应该如何服用二甲双胍？”

“分析一下王二的这份体检报告，他有哪些健康风险？”

“你叫什么名字？”、“今天天气怎么样？”

### 第二步，构建意图

不需要训练任何新模型，只需要在向 LLM 发送请求时，在 Prompt 中加入几个精心挑选的例子，让 LLM 自己来判断。

```
你是一个医疗问答意图识别专家。请判断以下用户问题的意图。意图只能是 【疾病知识问答, 药物信息查询, 诊疗指南检索, 患者病历分析, 闲聊】中的一种。

问题: "高血压有什么症状？"
意图: 疾病知识问答

问题: "阿司匹林有什么副作用？"
意图: 药物信息查询

问题: "你今天心情好吗？"
意图: 闲聊

---
现在，请判断以下这个新问题的意图：
问题: "{{用户的原始问题}}"
意图: 
```

### 第三步，改造派聪明

改造前的流程 : 用户问题 -> 混合搜索 -> LLM 生成答案

代码改造。

创建多个知识库索引 ：在 Elasticsearch 中，不再只有一个 knowledge 索引，而是根据资料类型创建多个索引，例如 medical_encyclopedia_idx , medication_manual_idx , clinical_guideline_idx 。

输入参数除了用户的 query ，还需要加上一个 target_index （目标索引）。

创建一个路由服务，首先调用意图识别模块，拿到意图标签后，根据预设的规则，决定调用 HybridSearchService 时应该传入哪个 target_index 。


![](https://files.mdnice.com/user/3903/3e58b6c0-8181-4fae-94d3-3ca2dd954b4c.jpg)


然后，在路由到这个分支时，代码会先调用一些定制化工具，将提取出的结构化数据和原始问题一起组织成更复杂的 Prompt，再交给 LLM。
