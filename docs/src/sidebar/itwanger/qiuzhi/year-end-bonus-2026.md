## 暑期实习经验分享

我最开始决定准备的时候是在2024年的5月中下旬。

当时我本来在纠结要不要准备软考，后来发现对我现阶段没什么用，于是在问了一个学长后决定把目标定在后端java了。本科的时候接触过一些web开发，但是从没有细细的学过，也没有去了解底层的深层原理。

从5月开始的准备就是从JavaWeb开始学的，然后重看mysql、redis、spring、springmvc、springboot、JUC、JVM，项目则是以[技术派为基础](https://mp.weixin.qq.com/s/No9F6sKhFnEeQ8H-GyzSCA)修改的 + SimpleDB（就是星球里的DB项目，但是用golang全部重写了）。


![](https://files.mdnice.com/user/3903/30aee6da-a406-4c6a-9c79-b6b8f1df899c.jpg)


目前拿到的是阿里云的offer，进去后面要转golang+k8s做云原生的开发。不过因此也许秋招时会多一条路，同时投传统后端开发+云原生的开发。

### 准备过程

八股部分：在去年的12月基本上过了第一遍，然后就去小红书实习了，后面八股其实没有再彻彻底底仔细的完全过了，都是随便翻翻看的方式去复习

刷题部分：力扣的hot100完完整整的做了2遍（包括所有困难题），后续又反反复复复习了两三遍；还是力扣的《面试经典150》大致做了一遍，一些比较偏的题没再看

### 面试经验

以下是一些面试经验的总结，具体的面试问题之前的帖子发过了

实习部分：

- 拷打实习做的项目的背景（非常高频）
- 我在其中做的主要工作，有什么意义（也非常高频）
- 我在其中遇到的困难是什么（很高频）
- 实习的项目和之前在学校做的有什么区别
- 在其中有没有遇到和其他同事合作出现矛盾的地方，如何协调解决的
- 项目中采用的技术有没有参考其他公司或者开源的一些技术，有没有现有的解决方案
- 项目中做的东西能不能引入AI或者AI Agent
- 写代码的过程中对AI工具的使用情况如何

具体的技术方面：会问到数据的持久化如何保证、系统接收的数据量级是怎样的如何保证不被打垮、有时候还会问到一些大数据的流批作业的处理方式（此时我如果实在答不上来会说这部分是大数据的不太了解）

总结：实习部分其实拷打的不是太狠，但是面试官会很感兴趣，这分担了拷打项目的压力以及高级八股的压力

### 技术派部分：

性能优化：是怎么做的压测，参数是多少，用的什么工具？为什么想到要做性能优化，有没有从其他的角度考虑优化策略，性能瓶颈的时候有没有使用arthas这样的工具或者OS级别的工具进行分析？

- 扫码登录的过程，如何保证登录的安全性，会不会有中间人攻击的问题
- 网站是如何上线的，如何配置nginx，证书的作用，如果扩容怎么办
- 项目中觉得最困难的点是什么，如何解决的
- 详细讲讲JVM优化的过程，进而可能讨论JVM的八股
- 讲讲异步优化的过程，为什么使用Rabbitmq
- 介绍下缓存优化的过程，自己优化的时候是怎么想的
- 介绍下在线状态人数统计的功能的优化过程，如果要支持分布式多机扩容的话那该怎么做

总结：整体围绕项目本身问，但是有一次运气不好，有个面试官说我的项目在其他地方见过很类似的，不知道是哪个球友也上线了，还和我面了同一家

### 心得
如果在找实习之前有一份中厂及以上的实习还是很有帮助的，能减少被拷打的压力

八股至少老老实实过一遍，建议有时间要自己研究下源码并且搞清原理，这样的话印象很少并且能和面试官battle

刷题至少把力扣hot100做一遍，面试80%是从hot100找的，剩下15%是hot100变种，刷熟了也能搞出来，最后的看命了现场死磕吧

笔试前先做一下ACM模式的输入输出，会减少到时候的时间

项目部分，如果能不临阵抱佛脚是最好的，面试官大多是想看到你在这个上面有自己的感悟思考，同时引到相关的八股上

在看别人的面经的时候，场景题部分切忌完全照搬，没有什么意义，针对项目的场景提问基本上每个面试官风格都不一样，而且会根据简历来，更建议的是针对面经中被问到的问题查漏补缺；然后自己面试中被问到了什么感觉很难答（就是给自己埋坑了），要及时更新简历，以防下次再被考住

写到这里，答案其实已经很明确了。

如果同学们能按照这位同学的心得经验去准备，暑期实习冲任何一家大厂都是完全没问题的。

如果你不知道投递哪些公司，那前面这两份榜单就是最好的参考。

有年终奖，至少差不了，大不了拿到offer后不去就是了。

冲。
