AI 圈最近是真热闹，各种 Agent 平台像雨后春笋一样冒出来，看得人眼花缭乱。

说实话，我之前一直是观望态度，毕竟这些平台多半还得自己配环境、搞 API，麻烦是其一，安全是其次。

但前两天，科大讯飞搞了个 AstronClaw，号称“部署只需 1-2 分钟”。我原本以为又是个“要折腾半天”的平台，结果实测后发现，这玩意儿是真的有点东西。

**AstronClaw 是什么**？

简单说，它是科大讯飞推出的 Agent 平台，主打一个字：“省”。省时间、省精力、省配置。而且它还是个“Sandbox 隔离”的平台，跑起来安全，不用担心数据泄露这种事。

如果你还在纠结选哪个 Agent 平台，或者被 OpenClaw 的部署流程搞到怀疑人生，这篇应该能帮你省点时间。

## 01、一键部署，真的是 1-2 分钟搞定

![](https://cdn.paicoding.com/stutymore/sucai-20260312221247.png)

我一开始就奔着部署体验去的。

点击“立即部署”，心里还想着至少得折腾个半天，结果 1 分多钟就搞定了。不用编写代码，不用额外设备，也不用担心环境依赖。

整个过程就是填几个表，点几下按钮，然后你的“电子龙虾”就上线了。干净又卫生，哦不，是安全又智能。

这一点确实加分。我知道很多小伙伴用 OpenClaw 踩过坑，不是环境配半天就是权限搞半天，最后还得找人救场。AstronClaw 把这一步简化成了“傻瓜式”，对新手友好，对老手省时间。

而且它是隔离运行在 Sandbox 中的，这一点让我放心不少。

之前用某些 Agent 平台，总觉得心里打鼓，搞不好就把自家代码或数据搞出去了。AstronClaw 的隔离机制，至少让你知道“这东西跑在独立空间里，乱不了”。

## 02、多模型自由切换，auto 模式省心

![](https://cdn.paicoding.com/stutymore/sucai-20260312221633.png)

模型这块，AstronClaw 给的配置还挺全。官方列表里有 Spark-X2、GLM-5、MiniMax 2.5、Kimi 2.5 这些国内一线模型。

但我觉得它最香的功能不是“多”，而是“智能”。auto 模式下，它可以根据任务需求自动切换模型，不用你手动来。

举个例子，你让它写代码，它可能自动切到 GLM-5 这种编程能力强的；你让它搞数据分析，它可能切到 Kimi 2.5 这种逻辑推理强的。你不用管，它会自己判断。

而且，最关键的是，它支持“无 API Key”模式。不用你去各个厂商注册账号、充值、搞配额，平台统一搞定。对于不想折腾多个账号、或者公司场景里不方便暴露个人账号的用户来说，这个设计真的贴心。

我实测了一下 auto 模式的智能化程度。除了前面说的任务路由、模型切换这些常见场景，我还试了几个“边界情况”：给它一个模糊的需求，看它能不能理解上下文；给它一个复杂的逻辑推理任务，看它会不会选择合适的模型；给它一个超出它能力范围的任务，看它是否会智能拒绝或者降级处理。

结果还是让我惊喜。auto 模式确实不是瞎切换，它会根据任务类型、复杂度、所需的专业能力来做判断。写代码选编程模型，搞数据分析选推理模型，写 Prompt 选理解模型。而且它的响应速度很快，基本上不用等。

对于新手或者不想折腾的用户来说，auto 模式真的是“懒人神器”。

你不用管模型选择，也不用担心 API Key，就给它一个任务，它自己搞定。这种“开箱即用”的体验，对比那些要自己配置环境、自己注册账号的平台，体验上是降维打击。

## 04、Skills 生态：131 个现成技能包

![](https://cdn.paicoding.com/stutymore/sucai-20260312222110.png)

我觉得 AstronClaw 最核心的优势其实是 Skills 生态。点开 Skills 小图标，里面已经有 131 个成熟的技能包。

包括智能简历生成、OCR、播客生成器、一句话复刻这些。而且它还连接了 ClawHub 生态，能轻松调用 10000+第三方 Skills。

![](https://cdn.paicoding.com/stutymore/sucai-20260312222441.png)

如果你想装第三方 Skills，往下翻就行。鼠标轻轻一点，几乎秒安装。这一点确实比 OpenClaw 那种还得手动上传、手动配置的体验好太多。

而且它还支持你上传自己的 Skill，让它更懂你。这一点对开发者或者有自己定制需求的小伙伴很友好。比如你有自己的业务逻辑、自己的 Prompt 模板，打包成 Skill 一传，就能复用。

## 05、飞书接入，保姆级教程来了

![](https://cdn.paicoding.com/stutymore/sucai-20260312222648.png)

渠道这块，AstronClaw 依然是“御三家”：飞书、钉钉、企业微信。我这次试了飞书接入，官方给的教程是真的保姆级。

第一步，点击配置说明里的“飞书开放平台”，新建一个 AstronClaw 助手应用。

![](https://cdn.paicoding.com/stutymore/sucai-20260312222831.png)

第二步，添加机器人能力。这一步官方给的提示很清晰，照着点就行。

![](https://cdn.paicoding.com/stutymore/sucai-20260312223004.png)

第三步，进入权限管理，将以下配置批量导入。这一块我是真要点赞官方了，直接给你个 JSON 配置，无脑复制粘贴就行。

```json
{
  "scopes": {
    "tenant": [
      "aily:file:read",
      "aily:file:write",
      "application:application.app_message_stats.overview:readonly",
      "application:application:self_manage",
      "application:bot.menu:write",
      "cardkit:card:write",
      "contact:user.employee_id:readonly",
      "corehr:file:download",
      "docs:document.content:read",
      "event:ip_list",
      "im:chat",
      "im:chat.access_event.bot_p2p_chat:read",
      "im:chat.members:bot_access",
      "im:message",
      "im:message.group_at_msg:readonly",
      "im:message.group_msg",
      "im:message.p2p_msg:readonly",
      "im:message:readonly",
      "im:message:send_as_bot",
      "im:resource",
      "sheets:spreadsheet",
      "wiki:wiki:readonly"
    ],
    "user": ["aily:file:read", "aily:file:write", "im:chat.access_event.bot_p2p_chat:read"]
  }
}
```

![](https://cdn.paicoding.com/stutymore/sucai-20260312223228.png)

第四步，回到 AstronClaw 的消息渠道配置，把飞书的 APP ID 和 APP Secret 复制粘贴过去。

![](https://cdn.paicoding.com/stutymore/sucai-20260312223356.png)

第五步，点击“保存配置”。

![](https://cdn.paicoding.com/stutymore/sucai-20260312223653.png)

第六步，回到飞书的“事件与回调”，把长连接打开，把 AstronClaw 推荐的`im.message.receive_v1`接收消息事件添加上。

![](https://cdn.paicoding.com/stutymore/sucai-20260312223749.png)

官方还提醒把回调配置的长连接也打开，顺便把“卡片回调交互”添加上。这样飞书的交互体验会更加友好一点。

![](https://cdn.paicoding.com/stutymore/sucai-20260312224113.png)

然后就可以点击版本管理与发布，准备通过 AstronClaw 和飞书交互起来吧。

![](https://cdn.paicoding.com/stutymore/sucai-20260312224239.png)

进入飞书，找到我们的 AstronClaw 助手，随便发送一条消息。

![](https://cdn.paicoding.com/stutymore/sucai-20260312225155.png)

收到回复的话，就证明我们的飞书渠道打通了，可以开始愉快地养虾了。

![](https://cdn.paicoding.com/stutymore/sucai-20260312225254.png)

给咱们的虾起个名字吧，叫 AstronClaw 助手。他是平行世界里你的 AI 助手，帮你处理文件、写代码、搞分析，或者就是陪你聊聊天。

整个飞书接入流程，我是觉得官方做得真的很细致。

每一步都有截图，每一步都有说明，配置直接给 JSON 不用你猜。对于没弄过飞书开放平台的小伙伴来说，这份教程基本上是“照着点就不会错”的水平。

## 06、Skills 实战：简历生成功能

![](https://cdn.paicoding.com/stutymore/sucai-20260312222110.png)

说到了 Skills 生态，我就忍不住测试一下简历生成功能。这个功能其实挺实用的，尤其是对正在找工作的同学来说。

我随便输了一下自己的信息：姓名“沉默王二”，年龄 18，学校郑州大学，求职意向是 AI 应用开发工程师。然后点击一下简历生成。

![](https://cdn.paicoding.com/paicoding/9f36250d3ab488fd1b00935fcb70fb6d.jpg)

哇，生成速度是真的快。几秒钟，一份结构完整的简历就出来了。而且内容质量还不错，有个人基本信息、教育背景、专业技能这些。

我是试了个“Java 开发岗”的简历，把一些我常用的技术栈和项目经历填了进去。AstronClaw 根据我的信息，帮我生成了项目经历部分。我看了看，格式还挺标准，技能和项目分开写，条理清楚。

生成的简历里，项目经历这部分我是觉得最亮眼的。它不是简单罗列“做了什么”，而是用了“公式化”的表达。

比如：“编写并优化 Docker Compose 编排脚本，统一管理 MySQL、Redis、MinIO、Console Hub 及 Workflow Engine 等 5+个核心服务的依赖关系与健康检查，实现了‘一键拉起’开发环境，将本地环境搭建时间从小时级降低至 30 分钟以内。”

这种写法显得专业，而且有数据支撑。对于 HR 来说，一看就知道你真的做过这些东西，不是随便吹的。

当然，简历只是个起点，面试的时候还得自己能说清楚。但至少，有了这个“敲门砖”，面试成功的机会能大不少。而且对于在校生或者应届生来说，能快速生成一份拿得出手的简历，已经是巨大的帮助了。

## 07、Sandbox 隔离：安全可靠

![](https://cdn.paicoding.com/stutymore/sucai-20260312221514.png)

这一点我前面也提到过，但我觉得值得再强调一下，因为安全真的很重要。

AstronClaw 采用的是 Sandbox 隔离运行机制。简单说，就是你的所有对话、所有数据，都跑在一个独立的隔离环境里。平台方看不到你的具体内容，也无法访问。

这一点对于企业用户来说太重要了。尤其是你在处理敏感信息的时候，或者测试的时候会用到一些内部数据。用公开的 Agent 平台，你总是担心“万一泄露了怎么办”。

AstronClaw 的 Sandbox 隔离机制至少让你放心不少。

而且，它还有个“无数据留存”的承诺。官方明确说了，不会永久保存你的聊天记录，对话结束后就删除。这一点比那些偷偷把用户数据拿去训练模型的平台强太多。

我在测试的过程中故意输入了一些敏感信息，想看看会不会有泄露。测试完之后，我在官方页面确认了一下聊天历史，确实一条记录都没有。这一点让我用起来更放心。
