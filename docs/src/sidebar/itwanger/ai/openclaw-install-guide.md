## 01、先搞清楚你要装的是什么

很多人装 OpenClaw 之前，根本不知道它到底是干嘛的。

OpenClaw 是一个开源的 AI 代理平台，核心能力就一句话：用自然语言驱动工具完成任务。它不是那种只会回答问题的聊天机器人，而是真正能动手的 Agent。读写文件、执行命令、操控浏览器、处理邮件，这些它都能干。

让它爆火的原因是，它虽然工作在你本地电脑或者云服务器上，但可以通过飞书、钉钉、企业微信这些 IM 工具进行远程管理。你在飞书群里发一条消息，它就能帮你整理文档、抓取网页、生成代码、处理 Excel，甚至还能定时提醒你该摸鱼了。

OpenClaw 本身不具备独立的大语言模型推理能力，需要对接大模型才能听懂指令。支持的大模型很多，阿里云百炼、智谱 GLM、OpenAI、Anthropic 都可以。

![](https://files.mdnice.com/user/3903/6487dae0-abeb-412f-8e4f-4bcc0931de95.jpg)

## 02、官方安装器：一行命令搞定

如果你是开发者，或者至少不害怕终端，那官方安装是最推荐的方式。

在开始之前，先把该装的装好。Node.js 必须升级到 22 以上，这是硬性要求，低于 22 版本会报错。macOS 用户可以直接用 Homebrew：`brew install node@22`。装完后 `node -v` 验证一下，显示 `v22.x.x` 就没问题。Windows 用户建议用 WSL2，在 Linux 环境里装会更顺畅。

然后你还需要准备一个大模型的 API Key。我以智谱 GLM 为例，访问智谱开放平台，在 API Keys 页面创建一个 Key，复制保存好。有需要的小伙伴建议先买个 lite 版本的套餐试试，OpenClaw 烧 token 还是挺快的。

环境准备好后，打开终端，复制粘贴这一行命令：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

回车，等着就行。安装器会自动帮你下载 CLI、配置环境、运行新手引导。整个过程大概五到十分钟，取决于你的网速。

![](https://cdn.paicoding.com/paicoding/122a4ac1028744398bb1cb2abee36fc0.png)

我第一次执行的时候卡死在了 `npm install failed`，直接 Ctrl-C 结束重新起了一个终端窗口再执行就成功了。这种小问题很常见，不用慌。

安装完成后，运行这个命令检查一下：

```bash
openclaw doctor
```

如果一切正常，你会看到所有检查项都是绿色的对勾。

![](https://cdn.paicoding.com/paicoding/210c27f919c1c99619371c57897784d6.png)

接下来是启动配置向导。选择 local 本地网关，然后是 AI 模型认证，把你准备好的 API Key 填进去。模型保持最新的 GLM-5 就可以了。

![](https://cdn.paicoding.com/paicoding/704b5b1ce0bda8de8bf0b7a35c025417.png)

然后进入 IM 的配置，选择飞书。此时会提示你接入飞书的配置信息，需要去飞书开放平台创建一个应用，获取 App ID 和 App Secret。这部分后面细说。

配置完成后启动服务：`openclaw gateway start`。默认监听 18789 端口。浏览器访问 `http://127.0.0.1:18789/`，如果能打开 OpenClaw 的 Web 控制面板，说明本地部署成功了。

## 03、AI Code Agent 代装

自己装太麻烦？那就让 AI 帮你装 AI 吧。

现在的 Coding Agent 选择很多，Claude Code、TRAE 都行。具体操作分三步。

第一步，去 Kimi、MiniMax、智谱这些大模型厂商的开放平台注册账号，充点钱，拿到你的 API 密钥。

第二步，打开 Claude Code，把 OpenClaw 的项目地址给它，然后发一条指令：这个 OpenClaw 项目帮我部署一下，对接智谱 GLM-5 的 API，密钥是 sk-xxxxx。

第三步，看着它干活就行。它会自动读取项目文件、分析依赖、配置环境、填入参数。大部分情况下，这一步它能自己搞定。

![](https://files.mdnice.com/user/3903/541e9c4b-89a2-4a04-8e7d-7a8381f593ec.png)

稍微麻烦一点的是飞书或钉钉的配置。你需要去飞书开放平台或钉钉开放平台注册一个应用，配置好权限，拿到 App ID 和 App Secret。然后让 Claude Code 帮你安装对应的插件并完成对接。

我用这个方式帮三个完全不懂代码的朋友装好了 OpenClaw。最快的一个，从零到飞书里跟机器人对话，前后不到二十分钟。说真的，让 AI 装 AI，这个体验还挺魔幻的。

## 04、阿里云一键部署

如果你连终端都不想碰，那阿里云的一键部署方案可能是你的救星。

阿里云轻量应用服务器直接提供了 OpenClaw 的镜像，配置要求也不高：2vCPU + 2GB 内存，端口 18789。操作流程大概是这样的：登录阿里云控制台，搜索 OpenClaw；选择轻量应用服务器，点击一键购买；等待镜像部署完成；获取阿里云百炼的 API-Key；进入服务器控制台，完成激活配置；生成专属 Token，访问 Web 界面。

全程可视化操作，不需要敲一行代码。

> https://www.aliyun.com/benefit/scene/moltbot

![](https://files.mdnice.com/user/3903/4df70e36-1666-4e9a-9a49-ced8d03ef196.jpg)

阿里云还贴心地提供了钉钉、飞书、企业微信、QQ 等多平台的接入教程。你想接哪个平台，跟着文档一步步来就行。

这种方式的缺点是需要花钱租服务器，一个月大概几十块钱。但比起自己折腾环境，这个成本完全可以接受。而且部署在云端还有个好处：不用担心本地电脑权限太大被 `rm -rf` 了。

## 05、各大厂云端版本：开箱即用

如果你连服务器都不想管，那可以试试各大厂推出的云端版本。

Coze Claw 是字节跳动出品，直接在 Coze 平台上就能用。优点是完全不用管部署，缺点是定制化程度低。百度云 Claw 是百度智能云推出的一站式解决方案，集成了文心一言的能力。如果你已经在用百度的云服务，这个方案集成起来很方便。Max Claw 是 MiniMax 官方的托管版本，直接对接 MiniMax 的模型 API，响应速度快。Kimi Claw 是月之暗面推出的版本，集成了 Kimi 的长文本能力，如果你经常处理长文档，这个版本值得一试。

![](https://files.mdnice.com/user/3903/fdc3aff9-e97a-4b5a-83ca-dce97b717c48.jpg)

这些云端版本的共同特点是：注册就能用，不用管部署，不用担心服务器宕机。但缺点也很明显：数据存在别人的服务器上，隐私性不如自建；定制化程度有限，不能随便改源码；API 调用量大了之后，费用可能会超出预期。

如果你只是想体验一下 OpenClaw 的能力，或者对隐私要求不高，云端版本是最省事的选择。

## 06、替代方案：国产版 OpenClaw

OpenClaw 虽然火，但毕竟是海外项目，有读者点名要国产版。

![](https://files.mdnice.com/user/3903/b64a09c7-6bfc-4b83-a099-4d10362a8117.png)

国内也涌现出了不少替代方案。阿里的 CoPaw 是阿里云推出的国产版 OpenClaw，深度集成了通义千问，三行代码就能部署，飞书、QQ、钉钉全打通。如果你追求稳定性和本地化支持，这个方案很靠谱。

> 推荐安装教程：https://paicoding.com/article/detail/2606200018087937

ZeroClaw 是社区驱动的开源替代方案，主打轻量化和易部署。PicoClaw 是另一个社区项目，特点是安装包特别小，资源占用低。这些替代方案的共同点是：针对国内网络环境做了优化，文档是中文的，社区支持也更及时。

不过说实话，如果你有能力装原版 OpenClaw，我还是建议装原版。毕竟生态最完善，更新也最快。

## 07、飞书接入完整流程

装好 OpenClaw 只是第一步，真正好用的是把它接入到你日常用的聊天工具里。这里以飞书为例，详细说一下接入流程。

首先进入飞书开放平台，创建一个企业自建应用，填写名称和描述。应用创建成功后，在凭证与基础信息页面能看到 App ID 和 App Secret，把这两个值复制保存好。

然后添加机器人能力。在应用的应用功能页面，点击添加应用能力，选择机器人。开通后，这个应用就能以机器人的身份出现在飞书群里了。

接下来配置权限。在权限管理页面，开通以下权限：`im:message` 获取与发送单聊、群聊消息；`im:message:send_as_bot` 以应用身份发消息；`im:chat` 获取群组信息。或者直接选择批量导入按钮，把 OpenClaw 官方推荐的权限全部接入进去。

![](https://cdn.paicoding.com/paicoding/fd3afd492d02756e40a5cf7548e56bdd.jpg)

然后配置事件订阅。开启事件订阅，选择长链接，添加事件 `im.message.receive_v1` 接收消息。这样当有人在飞书群里@机器人时，飞书会把消息推送到 OpenClaw。

![](https://cdn.paicoding.com/paicoding/26fdc24e8c98961ca471bfd17456e3d6.png)

配置完成后提交审核。飞书的审核比腾讯的 QQ 和企业微信方便多了，基本上免审。审核通过后，应用就可以在企业内使用了。

最后是配对。在飞书里打开应用，@它发一条消息，首次会提示你要配对，直接把这条消息发送到 OpenClaw 聊天窗口。配对完成后，再回到飞书随便发送一条信息，就完成通信了。

![](https://cdn.paicoding.com/paicoding/f807354d0468ea6a24c512384715908e.jpg)

## 08、常见问题排查

接入过程中可能会遇到一些问题，这里把最常见的情况列出来。

飞书响应很慢的话，可能是权限不够。直接在飞书这里添加通讯录基本信息的只读权限，会快很多。

OpenClaw 服务启动失败，可能原因有几个：Node.js 版本低于 22；端口被其他进程占用；API Key 配置错误。用 `node -v` 检查版本，`lsof -i:18789` 检查端口占用，`openclaw logs` 查看服务日志。

模型调用失败，通常是 API Key 无效或额度用尽。重新检查 API Key 是否正确，登录大模型平台确认额度是否充足。

![](https://cdn.paicoding.com/paicoding/ab0d8df7029fe0c06da466bf02263ed1.png)

## 09、付费上门安装

如果你实在是懒得折腾，或者折腾了好几天都没搞定，那还有一个选择：花钱请人帮你装。

![](https://files.mdnice.com/user/3903/a77427df-dfd5-4336-8b17-6a31fa14acd6.png)

海外的 SetUpClaw 网站推出了付费上门安装服务，托管服务 3000 美元，送 Mac Mini 的安装远程 5000 美元，现场安装 6000 美元。说实话，这个价格确实不便宜。但如果你是企业用户，需要一个稳定可靠的 AI 助手系统，这个投资可能是值得的。

国内小红书上已经出现了各种代安装服务，价格通常是 500-1000 元左右，全国各地都有人接单。我有个朋友就是在小红书上找人装的，花了 600 块，对方远程帮他搞定了全部配置，还顺带教了他怎么用。他觉得这钱花得值，因为他自己折腾了两天都没搞定。

当然，找陌生人远程操作你的电脑，安全性是个问题。建议找评价高、交易记录多的服务商，最好能走平台担保。
