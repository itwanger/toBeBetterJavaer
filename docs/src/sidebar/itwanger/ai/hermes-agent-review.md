## 01、v0.16.0 最重磅的更新：桌面应用

上次用 v0.9.0 的时候，Hermes 还是纯命令行工具。虽然有 CLI 的 TUI 界面，但说实话，对非技术用户不太友好。

v0.16.0 直接推出了原生桌面应用，基于 Electron 开发，支持 macOS、Linux、Windows 三大平台。

**一键安装，内置自动更新**。不需要手动拉代码、配环境、装依赖，下载安装包双击就能用。更新也是自动的，检测到新版本直接在应用内升级，不用再去 GitHub 下载。

【此处插入桌面应用主界面截图：截图目标：展示桌面应用的界面设计；关键词：Electron、聊天界面、模型选择器；建议位置：桌面应用窗口】

**拖拽文件直接进对话**。以前想让 Hermes 分析一个文档，得先把文件路径复制粘贴进去，现在直接拖进聊天框就行。支持多文件同时拖拽，体验和用微信发文件一样顺滑。

**状态栏模型切换器**。窗口底部有个模型选择器，点一下就能切换模型，不需要退出对话、不需要敲命令。支持模糊搜索，输入"sonnet"直接跳到 Claude Sonnet 4.6，输入"gpt"直接跳到 GPT-4o。

【此处插入模型切换器截图：截图目标：展示模型选择功能；关键词：模糊搜索、模型列表、快速切换；建议位置：桌面应用状态栏】

**多窗口并行会话**。可以同时开多个聊天窗口，每个窗口用不同的模型、不同的配置。比如一个窗口用 Claude Opus 4.8 做深度推理，另一个窗口用 Llama 3.3 跑本地测试，两边互不干扰。

**远程 Hermes 网关连接**。桌面应用不仅能连本地的 Hermes 实例，还能通过 OAuth 或用户名密码连接远程部署的 Hermes。这意味着你可以把 Hermes 跑在服务器上，然后用桌面应用远程操控，数据不落地本机。

对于企业用户来说，这个功能价值很大。可以把 Hermes 部署在内网服务器上，团队成员用桌面应用统一接入，既保证了数据安全，又方便管理。

## 02、Web Dashboard 升级成完整管理面板

v0.9.0 的时候，Hermes 新增了 Web Dashboard，但功能比较基础，主要就是查看对话历史和配置文件。

v0.16.0 把它升级成了完整的 Admin Panel，功能多了一大截。

**MCP 目录管理**。可以在网页上直接浏览、安装、卸载 MCP 服务器。以前装个 MCP 得去命令行敲 `hermes mcp install`，现在点几下鼠标就搞定。

界面上会显示每个 MCP 服务器提供的工具列表、参数说明、使用示例，不用再去翻文档。装完之后还能看到调用统计——这个 MCP 被调用了多少次、成功率多少、平均耗时多久，数据一目了然。

【此处插入 MCP 管理界面截图：截图目标：展示 MCP 目录和工具列表；关键词：MCP 服务器、工具管理、安装界面；建议位置：Web Dashboard】

**消息通道配置**。Telegram、Discord、Slack、飞书、企微这些消息平台，现在可以在网页上可视化配置了。填写 App ID、Secret、Webhook URL，测试连接，开通权限，整个流程都在浏览器里完成。

以前配飞书的时候，我得去 `.env` 文件里一行行改，写错一个字符就得重新来。现在有表单校验，填错了直接提示，省了不少排查时间。

**凭证和 Webhook 管理**。所有的 API Key、OAuth Token、Webhook URL 都能在这里统一管理。支持加密存储，支持权限控制，多人协作的时候不会出现"谁改了配置导致服务挂了"的情况。

【此处插入凭证管理界面截图：截图目标：展示凭证列表和权限设置；关键词：API Key 管理、加密存储、权限控制；建议位置：Web Dashboard】

**一键系统控制**。启动、停止、重启 Hermes 服务，查看日志，清理缓存，全部可以在网页上操作。对于不熟悉命令行的用户来说，这个功能降低了很多使用门槛。

我现在的用法是：日常对话用桌面应用，配置管理用 Web Dashboard，调试问题的时候才去 CLI。三个界面各司其职，体验比之前好太多了。

## 04、Quick Setup via Nous Portal

v0.16.0 新增了一个对新手特别友好的功能：Quick Setup。

以前装 Hermes，配置模型、配置工具、配置消息网关，得花半小时到一小时。每个环节都要去查文档，填配置文件，测试连通性。

现在如果你订阅了 Nous Portal，直接执行：

```bash
hermes setup --portal
```

OAuth 登录一次，所有配置自动完成。

**自动配置模型接入**。Nous Portal 提供 300+ 模型的统一接入，包括 OpenAI、Anthropic、Google、Meta、Mistral 等主流厂商。Quick Setup 会自动拿到你的 Portal Token，配置好所有模型的认证信息。

**自动配置 Tool Gateway**。Nous Portal 不只提供模型，还提供搜索引擎、图像生成、TTS、云浏览器这些工具服务。Quick Setup 会把这些工具的 API 端点和认证信息全部配好，装完就能用。

**自动配置消息通道**。如果你在 Nous Portal 里绑定了 Telegram、Discord 账号，Quick Setup 会自动把这些消息通道的配置同步过来，不需要手动填 Webhook URL 和 Token。

【此处插入 Quick Setup 执行截图：截图目标：展示一键配置过程；关键词：hermes setup --portal、OAuth 登录、自动配置；建议位置：终端命令行】

整个过程不到 5 分钟。对比之前手动配置的繁琐程度，这个体验提升真的是质的飞跃。

当然，Quick Setup 需要订阅 Nous Portal。如果你不想订阅，依然可以用传统方式手动配置 OpenRouter、OpenAI 或者自己部署的模型。

## 06、上下文压缩再优化

上次写 v0.9.0 的时候，重点讲了 Hermes 的四阶段上下文压缩。v0.16.0 在这个基础上又做了几个优化。

**压缩触发阈值可配置**。v0.9.0 的时候压缩阈值是硬编码的 50%，现在可以在配置文件里自己设置。如果你用的是 Claude Opus 4.8 的 1M 上下文，可以把阈值设高一点，比如 70%，减少压缩频率。

**摘要质量提升**。v0.16.0 换了一个更好的 prompt 模板，生成的摘要更结构化，信息密度更高。我对比了一下 v0.9.0 和 v0.16.0 的压缩摘要，新版本的摘要明显更准确，关键信息丢失得更少。

**/undo 命令支持批量撤回**。这个功能挺实用。如果你发现最近几轮对话跑偏了，可以用 `/undo 5` 一次性撤回最近 5 轮，重新开始。配合上下文压缩，可以更灵活地管理对话历史。

## 10、v0.16.0 相比 v0.9.0 的核心变化总结

两个月时间，从 v0.9.0 到 v0.16.0，Hermes Agent 的变化可以用"脱胎换骨"来形容。

**产品化程度大幅提升**。v0.9.0 还是开发者工具的定位，主要用户是懂命令行、会改配置文件的技术人员。v0.16.0 有了桌面应用、可视化管理面板、一键配置，非技术用户也能上手。

**技能生态从零到一**。v0.9.0 的技能主要靠自己生成或手动安装。v0.16.0 内置 19932 条技能目录，覆盖了绝大部分常见场景，不需要从零开始积累。

**上下文管理更成熟**。v0.9.0 的压缩机制已经比大部分 Agent 框架好了，但还有优化空间。v0.16.0 的压缩质量、触发策略、配置灵活性都有明显改进，长对话的体验好了一大截。

**国际化支持完善**。v0.16.0 的简体中文翻译，对国内用户来说是实实在在的便利。以前排查问题得先翻译报错信息，现在直接中文提示，省了不少时间。

**稳定性和安全性增强**。64 个 Bug 的修复，安全漏洞的修补，这些都是生产环境必须具备的条件。v0.9.0 适合尝鲜和测试，v0.16.0 可以考虑用在实际项目里了。

如果你还在用 v0.9.0 或更早的版本，强烈建议升级到 v0.16.0。升级成本很低，收益很大。

## 11、回顾：Hermes Agent 的核心能力

写到这里，新读者可能会问：Hermes Agent 到底是做什么的？核心能力是什么？

简单回顾一下（详细内容可以看我之前的两篇文章）。

**自主学习技能系统**。Hermes 会从你交给它的任务中自动提炼技能，用得越多越聪明。比如你让它做竞品分析，第一次可能需要 10 分钟，调用一堆工具。完成后它会生成一个技能文件，下次遇到类似任务直接调用这个技能，3 分钟搞定。

技能还会自我优化。如果某次执行失败了，或者你给了反馈，Hermes 会更新技能脚本，下次执行就是改进版。

**三层记忆系统**。第一层是会话搜索，基于 FTS5 全文索引，你提到"上次那个项目"它能准确找到。第二层是跨会话知识召回，关键信息会提炼成记忆条目，新会话自动加载。第三层是用户建模，通过 Honcho 观察你的使用习惯，逐步建立你的"用户画像"。

**多平台无缝协作**。CLI、桌面应用、Web Dashboard、Telegram、Discord、Slack、飞书、企微，所有平台的对话是连续的。你在电脑上聊到一半，出门切换到手机，接着之前的话题继续，不会出现"换个平台就得重新交代背景"的情况。

**并行分治能力**。复杂任务可以拆给多个子 Agent 并行执行。比如你让它调研三个竞品，它会启动三个子 Agent 同时干活，总耗时等于单个任务的时间，比串行快三倍。

**定时任务调度**。内置 cron 调度器，可以执行无人值守任务。每天定时抓取数据、定时生成报告、定时监控服务状态，都可以自动化。

这些核心能力在 v0.16.0 里都保留了，而且因为桌面应用、技能目录、上下文优化这些更新，用起来比之前顺手多了。

## 09、实际使用中的几个坑

说点实在的，Hermes Agent 不是完美的，用下来也有几个需要注意的地方。

**第一个坑是初始配置复杂**。虽然有 Quick Setup，但如果你不用 Nous Portal，想自己配 OpenRouter 或者 OpenAI API，需要手动设置的地方还是挺多的。尤其是消息网关，配置 Telegram、Discord 这些平台的 webhook，对新手不太友好。

我的建议是：如果你只是想试试 Hermes Agent 的能力，直接订阅 Nous Portal，用 Quick Setup 一键配置。如果你想深度定制，花点时间看看官方文档，按步骤来，别跳过。

**第二个坑是技能质量参差不齐**。Hermes Agent 自动生成的技能，有些很好用，有些就是把工具调用流程记录下来，没有实质性的优化。你需要定期去技能目录里看看，把不好用的技能删掉或者手动改进。

好消息是技能文件就是普通的 Python 脚本，你可以直接编辑。如果你懂点 Python，完全可以自己优化技能逻辑。

**第三个坑是记忆系统有时候会"记错"**。比如我有次调研两个项目，Hermes Agent 把项目 A 的特性记到了项目 B 的记忆条目里，导致后面给我的回答有点混乱。

这个问题可以通过定期清理记忆库来缓解。执行 `hermes memory` 可以查看和编辑记忆条目，把错误的记忆删掉就行。

**第四个坑是无服务器部署的冷启动时间**。虽然 Modal 和 Daytona 支持休眠/唤醒，但冷启动需要 5-10 秒，对于需要快速响应的场景不太适合。如果你是重度用户，还是建议用常驻的部署方式。

## 10、Hermes Agent vs Claude Code vs Codex

很多小伙伴会问：Hermes Agent 和 Claude Code、Codex 有什么区别？该用哪个？

我的看法是：**这三个工具的定位不同，不是替代关系，是互补关系**。

Claude Code 和 Codex 是专注于**代码生成和项目开发**的工具。它们的核心能力是理解代码上下文、执行 git 操作、运行测试、生成文档。适合写代码、调试、重构这些开发场景。

Hermes Agent 是专注于**任务自动化和知识管理**的工具。它的核心能力是技能沉淀、记忆系统、跨平台协作。适合调研、监控、定时任务、多轮对话这些场景。

我现在的工作流是：**写代码用 Claude Code，做调研用 Hermes Agent**。

比如我要给 PaiAgent 项目加一个新功能，技术方案的调研交给 Hermes Agent，具体的代码实现交给 Claude Code。两个工具配合使用，效率最高。

如果非要选一个，我的建议是：如果你主要做开发工作，选 Claude Code；如果你需要处理大量信息收集、定时任务、多平台协作，选 Hermes Agent。
